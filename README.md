# Business FAQ Bot — телеграм-ассистент для бизнеса на Claude

Телеграм-бот, который отвечает на частые вопросы клиентов по заданной базе
знаний. В демо — ассистент барбершопа: часы работы, цены, услуги, запись, адрес.

## Какую проблему решает

Администратор небольшого бизнеса (барбершоп, салон, студия, кафе) тратит время
на одни и те же вопросы в личке: «сколько стоит?», «до скольки работаете?»,
«как записаться?». Бот берёт эту рутину на себя:

- **снижает нагрузку на администратора** — типовые вопросы закрывает бот 24/7;
- **отвечает только по базе знаний бизнеса** и не выдумывает цены/условия;
- если ответа в базе нет — **не фантазирует**, а сообщает, что уточнит у
  администратора (безопасный fallback).

## Что делает бот

- Принимает вопрос клиента текстом и отвечает на основе базы знаний.
- Отвечает строго по базе: если данных нет — отдаёт заготовленную фразу-fallback,
  а не выдуманный ответ.
- `/admin` — демо-панель: показывает, как расширять базу знаний **без правки кода**.

## Стек

- **Python 3.12**
- **aiogram 3** — телеграм-бот (асинхронный)
- **OpenRouter** (OpenAI-совместимый API, `openai` SDK) — генерация ответов;
  модель по умолчанию `deepseek/deepseek-v4-flash`
- **PyYAML** — база знаний в отдельном YAML-файле
- **Docker / docker-compose** — деплой

## Как это устроено

База знаний лежит в [`data/knowledge_base.yaml`](data/knowledge_base.yaml) —
отдельно от кода. При старте вся база (она короткая, ~18 пунктов) целиком
вставляется в системный промпт модели вместе с инструкцией:

> «Отвечай только на основе базы знаний. Если ответа нет — скажи, что уточнишь у
> администратора, и не выдумывай факты».

Это простейший, но надёжный «RAG» для маленькой базы: модель видит все Q/A и
отвечает по ним. Для большой базы здесь появился бы поиск релевантных пунктов
перед запросом к модели — точка расширения отмечена в
[`app/knowledge_base.py`](app/knowledge_base.py).

Ответы генерирует LLM через **OpenRouter** (OpenAI-совместимый API) — модель
задаётся в `.env` в формате `provider/model`. Если API недоступен, бот не падает,
а отдаёт тот же вежливый fallback — см. [`app/llm.py`](app/llm.py).

## Расширение базы знаний без кода

1. Откройте [`data/knowledge_base.yaml`](data/knowledge_base.yaml).
2. Добавьте пункт в список `faq`:
   ```yaml
     - question: "Есть ли скидки студентам?"
       answer: "Да, по студенческому билету — скидка 10%."
   ```
3. Перезапустите бота (при Docker: `docker compose restart bot` — база
   примонтирована как volume, пересобирать образ не нужно).

## Запуск

### Вариант 1: Docker (рекомендуется)

1. Получите токен бота у [@BotFather](https://t.me/BotFather).
2. Получите ключ OpenRouter на [openrouter.ai/keys](https://openrouter.ai/keys).
3. Скопируйте `.env.example` в `.env` и заполните `BOT_TOKEN` и `OPENROUTER_API_KEY`:
   ```bash
   cp .env.example .env
   ```
4. Запустите:
   ```bash
   docker compose up -d --build
   ```

Логи: `docker compose logs -f`. Остановка: `docker compose down`.

### Вариант 2: локально

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env             # заполните BOT_TOKEN и OPENROUTER_API_KEY
python -m app.main
```

## Переменные окружения

| Переменная            | Назначение                                   | По умолчанию               |
|-----------------------|----------------------------------------------|----------------------------|
| `BOT_TOKEN`           | Токен бота от @BotFather (обязательно)        | —                          |
| `OPENROUTER_API_KEY`  | Ключ OpenRouter (обязательно)                 | —                          |
| `LLM_MODEL`           | Модель в формате `provider/model`             | `deepseek/deepseek-v4-flash` |
| `OPENROUTER_BASE_URL` | Базовый URL OpenRouter                         | `https://openrouter.ai/api/v1` |
| `KNOWLEDGE_BASE_PATH` | Путь к YAML с базой знаний                     | `data/knowledge_base.yaml` |

> Модель можно поменять на любую доступную в OpenRouter, задав `LLM_MODEL`
> (например, другой провайдер/модель).

## Ограничения

- Бот отвечает только по заданной базе знаний; он не заменяет живого
  администратора для нестандартных запросов (запись, жалобы) — на такие случаях
  сделан fallback «уточню у администратора».
- Диалог без памяти между сообщениями (каждый вопрос обрабатывается отдельно) —
  для FAQ этого достаточно; историю чата можно добавить, передавая предыдущие
  сообщения в `messages`.
